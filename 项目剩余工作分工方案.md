## 1. 项目整体规划回顾

### 1.1 完整的 5 阶段规划

根据项目文档（`PROJECT_COMPLETE.md`、`README.md`、`项目分析报告_LlamaIndex与RAG技术评估.md`），项目分为 5 个阶段：

```
Phase 0-1: 数据获取 (Data Ingestion)
    ↓
Phase 1: 向量化 (Embeddings)
    ↓
Phase 2: 主题建模 (BERTopic Clustering)
    ↓
Phase 3: FastAPI 后端 (Backend API)
    ↓
Phase 4: Streamlit 前端 (Frontend UI)
    ↓
Phase 5: RAG 对话 + 工程化 (RAG & Production)
```

### 1.2 项目核心功能

根据老师的需求：
> "用户输入标题、自己的见解或自然语言（可选），然后输出不同主题的论文摘要、标题等供用户参考，用户可以自行选择主题查看或者下载论文。"

**核心功能模块：**
1. **数据准备**：预先下载大量论文（本地论文库）
2. **主题发现**：使用 BERTopic 自动聚类
3. **后端 API**：提供论文查询、主题管理接口
4. **前端 UI**：用户界面（由其他成员负责）
5. **RAG 对话**：基于论文的智能问答（可选）

### 3.3 RAG 对话流程（待实现）

**基于本地预下载的论文库进行智能问答。**

```
用户选择主题（可选）
        ↓

        ↓
前端调用：POST /api/chat/message
        ↓
后端 RAG 处理：
┌─────────────────────────────────────────────────────────┐
│ 1. 向量检索（ChromaDB）                                  │
│    - 将问题转换为向量                                     │
│    - 在本地向量库中检索相似论文                           │
│    - 支持按主题过滤                                       │
│    - 返回 Top-K 相关论文（如 Top-3）                      │
│                                                         │
│ 2. 上下文拼接                                            │
│    - 提取检索到的论文摘要                                 │
│    - 拼接为上下文：论文1摘要 + 论文2摘要 + 论文3摘要      │
│                                                         │
│ 3. LLM 生成答案（OpenAI API）                            │
│    - Prompt: 系统提示 + 上下文 + 用户问题                │
│    - 调用 GPT-3.5/4 生成答案                             │
│                                                         │
│ 4. 返回结果                                              │
│    - 答案文本                                            │
│    - 来源论文列表（标题、作者、PDF 链接）                 │
└─────────────────────────────────────────────────────────┘
        ↓
前端显示答案和来源论文
```

**关键点：**
- ✅ 所有检索都在本地完成（ChromaDB 本地向量库）
- ✅ 不需要实时从 arXiv 下载论文
- ✅ 只需要调用 OpenAI API 生成答案（需要联网）
- ✅ 答案基于预下载的论文内容，可追溯来源

## 4. 数据流程详解

### 4.1 数据类型和格式

#### 数据类型 1：原始论文数据（CSV）

**文件位置：** `data/raw/arxiv_*.csv`

**数据结构：**
```csv
id,title,abstract,authors,published,updated,categories,url,pdf_url,comment
http://arxiv.org/abs/2301.00001v1,"Example Paper Title","This is the abstract...","['Alice Smith', 'Bob Johnson']",2023-01-01 00:00:00,2023-01-02 00:00:00,"['cs.CL', 'cs.AI']",http://arxiv.org/abs/2301.00001v1,http://arxiv.org/pdf/2301.00001v1.pdf,
```

**字段说明：**
| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | str | arXiv 论文 URL | `http://arxiv.org/abs/2301.00001v1` |
| `title` | str | 论文标题 | `"BERT: Pre-training..."` |
| `abstract` | str | 论文摘要（核心数据） | `"We introduce a new..."` |
| `authors` | str (list) | 作者列表（字符串格式） | `"['Alice', 'Bob']"` |
| `published` | datetime | 发布日期 | `2023-01-01 00:00:00` |
| `updated` | datetime | 更新日期 | `2023-01-02 00:00:00` |
| `categories` | str (list) | arXiv 分类 | `"['cs.CL', 'cs.AI']"` |
| `url` | str | 论文页面链接 | `http://arxiv.org/abs/2301.00001v1` |
| `pdf_url` | str (可选) | PDF 下载链接 | `http://arxiv.org/pdf/2301.00001v1.pdf` |

**数据量：** 每个 CSV 文件约 1000-1500 篇论文

---

#### 数据类型 2：向量数据（.npy）

**文件位置：** `data/processed/*_embeddings.npy`

**数据结构：**
```python
# NumPy 数组
shape: (N, D)
  N = 论文数量（如 1000）
  D = 向量维度（768 for all-mpnet-base-v2）

# 示例
embeddings = np.load("medical_embeddings.npy")
print(embeddings.shape)  # (1000, 768)
```

**说明：**
- 每篇论文的摘要被转换为一个 768 维的向量
- 向量捕捉了摘要的语义信息
- 相似的论文在向量空间中距离较近

**文件大小：** 约 6-10 MB（取决于论文数量）

---

#### 数据类型 3：主题模型（BERTopic）

**文件位置：** `models/*_topic_model/`

**文件结构：**
```
models/medical_topic_model/
├── config.json                  # BERTopic 配置
├── topics.json                  # 主题信息
├── topic_embeddings.npy         # 主题向量
├── ctfidf_model.pkl             # c-TF-IDF 模型
├── umap_model.pkl               # UMAP 降维模型
├── hdbscan_model.pkl            # HDBSCAN 聚类模型
└── vectorizer_model.pkl         # 词向量化模型
```

**关联的 CSV 文件：** `models/medical_topic_model_papers.csv`

**CSV 新增字段：**
```csv
...,topic_id,topic_name
...,0,"medical_imaging_diagnosis"
...,1,"drug_discovery_molecular"
...,2,"genomics_variant_calling"
```

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `topic_id` | int | 主题 ID（-1 表示噪声） | `0`, `1`, `2`, `-1` |
| `topic_name` | str | 主题名称（自动生成） | `"medical_imaging_diagnosis"` |

---

#### 数据类型 4：ChromaDB 向量数据库（待实现）

**文件位置：** `data/chroma/`

**数据结构：**
```python
# ChromaDB Collection
{
    "id": "paper_0",
```

**用途：** RAG 检索（根据用户问题找到相似论文）
